## Requirements

KoAkademy ships with sensible defaults that let you run locally without Docker or any external services.

### Required

| Dependency | Version | Purpose |
|---|---|---|
| PHP | 8.4+ | Application runtime (extensions: SQLite, PDO, mbstring, bcmath, zip, intl, gd, redis, exif) |
| Composer | 2.x | PHP dependency manager |
| Node.js | 22+ | Frontend asset build (not needed for the pre-built Docker image) |
| Database | — | SQLite (default), PostgreSQL 17, or MySQL 8.4 |

### Optional

| Service | Purpose | Fallback when not used |
|---|---|---|
| Redis 7+ | Cache, queues, sessions, Pulse ingest, Horizon | SQLite / database driver |
| Meilisearch | Full-text search via Laravel Scout | Collection / database driver |
| Mailpit / SMTP | Transactional email | Log driver |
| MinIO / R2 / S3 | Object storage | Local filesystem |
| Gotenberg | PDF rendering (headless browser) | Cloudflare Browser Rendering or DOMPDF |

## Get Started

KoAkademy is built to be **self-hosted**. The recommended production path uses our pre-built Docker image with Docker Compose.

### Self-host with Docker (recommended)

```bash "Terminal"
# 1. Create docker-compose.yml (see Installation page for the full file)
# 2. Copy the production env example
cp .env.production.example .env

# 3. Generate an app key
docker run --rm \
  -e APP_KEY=base64:000000000000000000000000000000000000000000= \
  yukazaki/koakademy:latest \
  php artisan key:generate --show

# 4. Start the stack
docker compose up -d

# 5. Create your admin user
docker exec -it koakademy-app php artisan make:filament-user
```

Visit `http://localhost:8000`. Full details: [Installation](/getting-started/installation).

### Local development (for contributors)

If you want to contribute code, you can run KoAkademy locally with just PHP and SQLite — no Docker required:

```bash "Terminal"
git clone https://github.com/yukazakiri/koakademy.git
cd koakademy
composer install
cp .env.example .env
touch database/database.sqlite
php artisan key:generate
php artisan migrate
npm install && npm run build
php artisan serve --host=0.0.0.0 --port=8000
```

This uses SQLite, database cache, sync queues, and log mail — zero external dependencies.
