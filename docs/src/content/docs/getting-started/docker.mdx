KoAkademy provides pre-built Docker images for both **quick evaluation** and **production deployment**. You can run it in seconds with a single `docker run` command, or deploy a full production stack with Docker Compose.

## Quick Run

Evaluate KoAkademy instantly with our Docker Hub image.

:::caution
The pre-built image is optimized for production-style deployments. For a quick evaluation, you must provide a valid Laravel `APP_KEY` and tell the container to use SQLite so it can start without an external database.
:::

### 1. Generate an app key

```bash "Terminal"
APP_KEY="base64:$(openssl rand -base64 32)"
echo "$APP_KEY"
```

On Windows (PowerShell):

```powershell
$bytes = New-Object byte[] 32
(New-Object Security.Cryptography.RNGCryptoServiceProvider).GetBytes($bytes)
$APP_KEY = "base64:" + [Convert]::ToBase64String($bytes)
Write-Output $APP_KEY
```

### 2. Run the container

```bash "Terminal"
docker run -d \
  --name koakademy \
  -p 8000:8000 \
  -e APP_KEY="$APP_KEY" \
  -e APP_ENV=local \
  -e DB_CONNECTION=sqlite \
  -e DB_DATABASE=/app/database/database.sqlite \
  -e CACHE_STORE=database \
  -e SESSION_DRIVER=database \
  -e QUEUE_CONNECTION=sync \
  -e MAIL_MAILER=log \
  yukazaki/koakademy:latest
```

Visit `http://localhost:8000`. The container runs FrankenPHP with Laravel Octane on port `8000`.

:::note
Data is stored inside the container. If you remove the container, the SQLite database and uploaded files are lost. Mount a volume for persistence:

```bash
  -v koakademy-sqlite:/app/database \
  -v koakademy-storage:/app/storage \
```
:::

:::tip
Use `yukazaki/koakademy:dev-latest` for the latest development build.
:::

## Docker Compose

For production or persistent deployments, use Docker Compose with PostgreSQL and Redis.

The project includes an `.env.production.example` file with sensible defaults. Start from there instead of writing an `.env` from scratch.

### 1. Create a Compose File

Create `docker-compose.yml`:

```yaml
services:
  app:
    image: yukazaki/koakademy:latest
    container_name: koakademy-app
    restart: unless-stopped
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      - pgsql
      - redis
    volumes:
      - koakademy-storage:/app/storage
    networks:
      - koakademy

  pgsql:
    image: postgres:18-alpine
    container_name: koakademy-pgsql
    restart: unless-stopped
    environment:
      POSTGRES_DB: koakademy
      POSTGRES_USER: koakademy
      POSTGRES_PASSWORD: replace-with-strong-password
    volumes:
      - koakademy-pgsql:/var/lib/postgresql
    networks:
      - koakademy

  redis:
    image: redis:7-alpine
    container_name: koakademy-redis
    restart: unless-stopped
    volumes:
      - koakademy-redis:/data
    networks:
      - koakademy

volumes:
  koakademy-storage:
  koakademy-pgsql:
  koakademy-redis:

networks:
  koakademy:
    driver: bridge
```

:::caution
If you are using PostgreSQL 18+, mount the volume to `/var/lib/postgresql` instead of `/var/lib/postgresql/data`. The PostgreSQL 18+ Docker image stores data in a version-specific subdirectory inside `/var/lib/postgresql` to support `pg_upgrade`.
:::

### 2. Create Your Environment File

Copy the production example and customize it:

```bash
cp .env.production.example .env
```

Edit the values that matter for your setup:

```ini
APP_NAME="KoAkademy"
APP_ENV=production
APP_DEBUG=false
APP_URL=http://localhost:8000
APP_KEY=   # leave empty for now; we will generate it below

ADMIN_HOST=admin.localhost
PORTAL_HOST=localhost

DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
DB_DATABASE=koakademy
DB_USERNAME=koakademy
DB_PASSWORD=replace-with-strong-password

CACHE_STORE=redis
QUEUE_CONNECTION=redis
SESSION_DRIVER=redis

REDIS_HOST=redis
REDIS_PORT=6379

MAIL_MAILER=log
MAIL_FROM_ADDRESS=noreply@koakademy.edu
MAIL_FROM_NAME="${APP_NAME}"

OCTANE_SERVER=frankenphp
OCTANE_HOST=0.0.0.0
OCTANE_PORT=8000
```

:::caution
`ADMIN_HOST` and `PORTAL_HOST` are used for domain-based routing in production. If you are evaluating on `localhost`, keep them simple or update your `/etc/hosts` file to match.
:::

Generate an app key and add it to `.env`:

```bash
docker run --rm \
  -e APP_KEY=base64:000000000000000000000000000000000000000000= \
  yukazaki/koakademy:latest \
  php artisan key:generate --show
```

Copy the output into `APP_KEY=` in your `.env`.

:::note
Migrations run automatically when the `app` container starts (`RUN_MIGRATIONS=true` by default). You only need to run them manually if you disable auto-migrations or want to verify the state.
:::

### 3. Start the Stack

```bash
docker compose up -d
```

### 4. Create an Admin User

```bash
docker exec -it koakademy-app php artisan make:filament-user
```

Follow the prompts to create your first administrator.

### 5. Verify Deployment

```bash
docker ps
docker logs koakademy-app --tail=200
docker exec koakademy-app php artisan about
```

## Production Compose with All Services

For a full production deployment with Meilisearch and Gotenberg:

```yaml
services:
  app:
    image: yukazaki/koakademy:latest
    container_name: koakademy-app
    restart: unless-stopped
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      - pgsql
      - redis
      - meilisearch
      - gotenberg
    volumes:
      - koakademy-storage:/app/storage
    networks:
      - koakademy

  pgsql:
    image: postgres:18-alpine
    container_name: koakademy-pgsql
    restart: unless-stopped
    environment:
      POSTGRES_DB: koakademy
      POSTGRES_USER: koakademy
      POSTGRES_PASSWORD: replace-with-strong-password
    volumes:
      - koakademy-pgsql:/var/lib/postgresql
    networks:
      - koakademy

  redis:
    image: redis:7-alpine
    container_name: koakademy-redis
    restart: unless-stopped
    volumes:
      - koakademy-redis:/data
    networks:
      - koakademy

  meilisearch:
    image: getmeili/meilisearch:v1.42
    container_name: koakademy-meilisearch
    restart: unless-stopped
    environment:
      MEILI_NO_ANALYTICS: "false"
    volumes:
      - koakademy-meilisearch:/meili_data
    networks:
      - koakademy

  gotenberg:
    image: gotenberg/gotenberg:8
    container_name: koakademy-gotenberg
    restart: unless-stopped
    networks:
      - koakademy

volumes:
  koakademy-storage:
  koakademy-pgsql:
  koakademy-redis:
  koakademy-meilisearch:

networks:
  koakademy:
    driver: bridge
```

Add to your `.env`:

```ini
SCOUT_DRIVER=meilisearch
MEILISEARCH_HOST=http://meilisearch:7700

LARAVEL_PDF_DRIVER=gotenberg
LARAVEL_PDF_PRODUCTION_DRIVER=gotenberg
GOTENBERG_URL=http://gotenberg:3000
```

:::note
The `app` container runs FrankenPHP, Horizon, Pulse, and Nightwatch together via Supervisor when `QUEUE_CONNECTION=redis`. You do **not** need separate `horizon` or `pulse` containers unless you are horizontally scaling workers across multiple hosts.
:::

## Scaling Workers Separately

If you need to scale Horizon or Pulse workers independently (for example, on a different host or with different resource limits), run dedicated worker containers:

```yaml
  horizon:
    image: yukazaki/koakademy:latest
    container_name: koakademy-horizon
    restart: unless-stopped
    env_file:
      - .env
    environment:
      CONTAINER_MODE: horizon
    depends_on:
      - pgsql
      - redis
    volumes:
      - koakademy-storage:/app/storage
    networks:
      - koakademy
```

Use `CONTAINER_MODE=horizon` to run only the Horizon queue worker, bypassing the web server entirely. The same pattern works for `scheduler` and `nightwatch`.

## Available Container Modes

The KoAkademy image supports different container modes via the `CONTAINER_MODE` environment variable:

| Mode | Use Case | Command |
|---|---|---|
| `http` (default) | Web server with Octane, Horizon, Pulse, and Nightwatch | `docker run ... yukazaki/koakademy:latest` |
| `horizon` | Queue worker with Horizon only | `docker run -e CONTAINER_MODE=horizon ...` |
| `scheduler` | Cron scheduler only | `docker run -e CONTAINER_MODE=scheduler ...` |
| `worker` | General queue worker only | `docker run -e CONTAINER_MODE=worker ...` |
| `nightwatch` | Nightwatch monitoring only | `docker run -e CONTAINER_MODE=nightwatch ...` |

## Rolling Updates

To update to the latest image:

```bash
docker compose pull
docker compose up -d
```

The container automatically runs migrations on startup. After the update, warm the caches:

```bash
docker exec koakademy-app php artisan optimize
```

## Backup & Rollback

Back up before every deploy:

```bash
# Database (pass the password via the environment)
docker exec -e PGPASSWORD=replace-with-strong-password koakademy-app \
  pg_dump -U koakademy koakademy > backup.sql

# Storage
docker run --rm -v koakademy-storage:/data -v $(pwd):/backup alpine \
  tar czf /backup/storage.tar.gz -C /data .
```

To roll back to a previous image tag:

```bash
# Update your docker-compose.yml to use the previous tag, then:
docker compose down
docker compose up -d
```
