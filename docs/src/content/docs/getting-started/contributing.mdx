Thank you for considering contributing to KoAkademy! We welcome bug reports, feature requests, documentation improvements, and code contributions.

## Reporting Issues

Before opening an issue, please:

1. Search existing issues to avoid duplicates.
2. Use the issue templates when available.
3. Provide a minimal reproduction for bugs.

## Development Setup

1. Fork and clone the repository.
2. Follow the [Installation](/getting-started/installation) guide for your OS.
3. Create a branch: `git checkout -b feature/your-feature-name`.

## Code Style

- PHP: Run `vendor/bin/sail bin pint --dirty` before committing.
- JavaScript/TypeScript: Run `vendor/bin/sail npm run lint`.
- Follow existing patterns in neighboring files.

## Testing

All changes should include tests:

```bash
vendor/bin/sail artisan test --compact tests/Feature/YourFeatureTest.php
```

## Pull Request Process

1. Ensure tests pass and code is formatted.
2. Update documentation if your change affects user-facing behavior.
3. Open a PR against the `main` branch with a clear description.
