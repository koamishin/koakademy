KoAkademy is built to be **self-hosted**. The primary installation path uses the pre-built Docker image with Docker Compose — production-ready in under 10 minutes. If you want to contribute code, local development paths are available below.

| Path | Best For | Time |
|---|---|---|
| [Docker Compose](#self-hosted-production) | Production self-hosting | 10 minutes |
| [Laravel Herd](#local-development) | macOS / Windows development | 5 minutes |
| [Native / Manual](#native-manual) | Linux or full-control dev setups | 15 minutes |
| [Docker Sail](#docker--laravel-sail) | Isolated contributor environment | 10 minutes |

## Self-Hosted (Production)

The recommended way to run KoAkademy in production is with Docker Compose using the pre-built image. No PHP, Composer, or Node.js required on the host — just Docker.

### Prerequisites

| Tool | Version |
|---|---|
| Docker | 24+ |
| Docker Compose | v2 |

### 1. Create `docker-compose.yml`

```yaml
services:
  app:
    image: yukazaki/koakademy:latest
    container_name: koakademy-app
    restart: unless-stopped
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      pgsql:
        condition: service_healthy
      redis:
        condition: service_healthy
    volumes:
      - koakademy-storage:/app/storage
    networks:
      - koakademy
    healthcheck:
      test: ["CMD", "healthcheck"]
      interval: 5s
      timeout: 5s
      retries: 10
      start_period: 15s

  pgsql:
    image: postgres:18-alpine
    container_name: koakademy-pgsql
    restart: unless-stopped
    environment:
      POSTGRES_DB: koakademy
      POSTGRES_USER: koakademy
      POSTGRES_PASSWORD: replace-with-strong-password
    volumes:
      - koakademy-pgsql:/var/lib/postgresql
    networks:
      - koakademy
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U koakademy -d koakademy"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

  redis:
    image: redis:7-alpine
    container_name: koakademy-redis
    restart: unless-stopped
    volumes:
      - koakademy-redis:/data
    networks:
      - koakademy
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

volumes:
  koakademy-storage:
  koakademy-pgsql:
  koakademy-redis:

networks:
  koakademy:
    driver: bridge
```

### 2. Create Your `.env` File

Copy the production example and customize it:

```bash
cp .env.production.example .env
```

The essential variables to set:

```ini
APP_NAME="KoAkademy"
APP_ENV=production
APP_DEBUG=false
APP_URL=http://localhost:8000
APP_KEY=   # generate in the next step

DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
DB_DATABASE=koakademy
DB_USERNAME=koakademy
DB_PASSWORD=replace-with-strong-password

CACHE_STORE=redis
QUEUE_CONNECTION=redis
SESSION_DRIVER=redis

REDIS_HOST=redis
REDIS_PORT=6379

MAIL_MAILER=log   # switch to smtp for real email
MAIL_FROM_ADDRESS=noreply@koakademy.edu
MAIL_FROM_NAME="${APP_NAME}"

OCTANE_SERVER=frankenphp
OCTANE_HOST=0.0.0.0
OCTANE_PORT=8000
```

:::note
`PORTAL_HOST` and `ADMIN_HOST` default to `localhost` if not set, so the app works immediately on `http://localhost:8000`. Set them explicitly when deploying to a real domain (e.g. `portal.yourschool.edu`).
:::

### 3. Generate an App Key

```bash
docker run --rm \
  -e APP_KEY=base64:000000000000000000000000000000000000000000= \
  yukazaki/koakademy:latest \
  php artisan key:generate --show
```

Copy the output into `APP_KEY=` in your `.env`.

### 4. Start the Stack

```bash
docker compose up -d
```

Migrations run automatically on first start. Check that everything is healthy:

```bash
docker ps --format "table {{.Names}}\t{{.Status}}"
docker logs koakademy-app --tail=50
```

### 5. Create an Admin User

```bash
docker exec -it koakademy-app php artisan make:filament-user
```

Follow the prompts to create your first administrator.

### 6. Verify Everything Works

```bash
# Health endpoint
curl -s http://localhost:8000/health

# Octane status
docker exec koakademy-app php artisan octane:status

# Full system overview
docker exec koakademy-app php artisan about
```

Your KoAkademy instance is now running. Visit `http://localhost:8000`.

## Local Development

These paths are for developers who want to contribute to KoAkademy, build modules, or run the app without Docker.

### Laravel Herd

[Laravel Herd](https://herd.laravel.com/) is the fastest non-Docker path on **macOS** and **Windows**. Herd provides PHP, Composer, Node, SSL, and automatic `.test` domain resolution.

**Windows automated setup:**

```powershell "Windows PowerShell"
git clone https://github.com/yukazakiri/koakademy.git
cd koakademy
.\scripts\dev-setup.ps1
```

The script sets up SQLite by default, installs dependencies, generates an app key, runs migrations, and builds frontend assets. If Docker is running, it can optionally start PostgreSQL, MySQL, Redis, or Mailpit.

**macOS manual setup:**

```bash "Terminal"
git clone https://github.com/yukazakiri/koakademy.git
cd koakademy
composer install
cp .env.example .env

touch database/database.sqlite
php artisan key:generate
php artisan migrate
npm install
npm run build
npm run dev
```

Herd automatically serves the project at a `.test` domain.

### Native / Manual

Works on any OS with PHP 8.4+, Composer, and Node.js 22+. Uses SQLite by default.

```bash "Terminal"
git clone https://github.com/yukazakiri/koakademy.git
cd koakademy
composer install
cp .env.example .env

touch database/database.sqlite
php artisan key:generate
php artisan migrate
npm install
npm run build
php artisan serve --host=0.0.0.0 --port=8000
```

Visit `http://localhost:8000`. To switch to PostgreSQL, see [Configuration](/getting-started/configuration#database-configuration).

### Docker / Laravel Sail

A fully containerized development stack with PostgreSQL, Redis, and Mailpit.

```bash "Terminal"
git clone https://github.com/yukazakiri/koakademy.git
cd koakademy
composer install
cp .env.example .env
vendor/bin/sail up -d
vendor/bin/sail npm install
vendor/bin/sail artisan key:generate
vendor/bin/sail artisan migrate
vendor/bin/sail npm run dev
```

Use service names inside Docker (not `127.0.0.1`):

```ini
DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
DB_DATABASE=koakademy
DB_USERNAME=sail
DB_PASSWORD=
REDIS_HOST=redis
MAIL_HOST=mailpit
```
