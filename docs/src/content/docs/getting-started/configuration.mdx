KoAkademy is configured through environment variables. Copy `.env.example` to `.env` and customize only what you need.

The local development defaults are intentionally lightweight:

```ini
DB_CONNECTION=sqlite
CACHE_STORE=database
QUEUE_CONNECTION=sync
MAIL_MAILER=log
```

That means PostgreSQL/MySQL, Redis, and Mailpit are **optional** for local development.

## Required Variables

| Variable | Description | Default |
|---|---|---|
| `APP_NAME` | Application display name | `KoAkademy` |
| `APP_ENV` | Environment name | `local` |
| `APP_KEY` | Encryption key | generated by `php artisan key:generate` |
| `APP_URL` | Canonical app URL | `http://localhost` |
| `ADMIN_HOST` | Admin panel host | `localhost` (falls back if unset) |
| `PORTAL_HOST` | Student portal host | `localhost` (falls back if unset) |
| `DB_CONNECTION` | Database driver | `sqlite` |
| `CACHE_STORE` | Cache driver | `database` |
| `QUEUE_CONNECTION` | Queue driver | `sync` |
| `MAIL_MAILER` | Mail driver | `log` |

## Generating an App Key

```bash
php artisan key:generate
```

For a Docker container:

```bash
docker exec koakademy-app php artisan key:generate
```

## Database Configuration

### SQLite (Default)

SQLite is the default for local development and requires no external service:

```ini
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

Create the database file:

```bash
touch database/database.sqlite
php artisan migrate
```

### PostgreSQL via Docker Run

```bash
docker run -d --name koakademy-pgsql \
  -p 5432:5432 \
  -e POSTGRES_DB=koakademy \
  -e POSTGRES_USER=koakademy \
  -e POSTGRES_PASSWORD=koakademy \
  -v koakademy-pgsql:/var/lib/postgresql \
  postgres:17-alpine
```

:::caution
If you are using PostgreSQL 18+, the mount point must be `/var/lib/postgresql` (not `/var/lib/postgresql/data`). The PostgreSQL 18+ Docker image stores data in a version-specific subdirectory to support `pg_upgrade`.
:::

Use `127.0.0.1` because the app runs on your host machine:

```ini
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=koakademy
DB_USERNAME=koakademy
DB_PASSWORD=koakademy
```

### MySQL via Docker Run

```bash
docker run -d --name koakademy-mysql \
  -p 3306:3306 \
  -e MYSQL_DATABASE=koakademy \
  -e MYSQL_USER=koakademy \
  -e MYSQL_PASSWORD=koakademy \
  -e MYSQL_ROOT_PASSWORD=root \
  -v koakademy-mysql:/var/lib/mysql \
  mysql:8.4
```

```ini
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=koakademy
DB_USERNAME=koakademy
DB_PASSWORD=koakademy
```

### Sail / Compose Databases

If the app itself is running inside Docker Compose, use service names instead of localhost:

```ini
DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
DB_DATABASE=koakademy
DB_USERNAME=sail
DB_PASSWORD=
```

:::caution
Use `127.0.0.1` only when PHP runs on your host machine, for example Herd or `php artisan serve`. Use service names like `pgsql`, `redis`, and `mailpit` when PHP runs inside Docker/Sail.
:::

## Cache and Queue

### Default: No Redis

```ini
CACHE_STORE=database
QUEUE_CONNECTION=sync
```

This is best for local development because no worker process is required.

### Optional Redis

Start Redis:

```bash
docker run -d --name koakademy-redis \
  -p 6379:6379 \
  -v koakademy-redis:/data \
  redis:7-alpine
```

Then update `.env`:

```ini
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
CACHE_STORE=redis
QUEUE_CONNECTION=redis
```

If you use Redis queues, run a worker or Horizon:

```bash
php artisan queue:work
# or
php artisan horizon
```

## Mail

### Default: Log Mail

```ini
MAIL_MAILER=log
```

Laravel writes mail to the log instead of sending it.

### Optional Mailpit

Start Mailpit:

```bash
docker run -d --name koakademy-mailpit \
  -p 1025:1025 \
  -p 8025:8025 \
  axllent/mailpit:latest
```

Then update `.env`:

```ini
MAIL_MAILER=smtp
MAIL_HOST=127.0.0.1
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
```

Mailpit UI: `http://localhost:8025`.

### SMTP (Production)

```ini
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailgun.org
MAIL_PORT=587
MAIL_USERNAME=postmaster@koakademy.edu
MAIL_PASSWORD=your-mailgun-key
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@koakademy.edu
MAIL_FROM_NAME="${APP_NAME}"
```

## PDF Rendering

KoAkademy supports multiple PDF drivers:

```ini
LARAVEL_PDF_DRIVER=cloudflare
LARAVEL_PDF_PRODUCTION_DRIVER=cloudflare
LARAVEL_PDF_PRODUCTION_FALLBACK=dompdf
LARAVEL_PDF_ROLLBACK_DRIVER=dompdf

CLOUDFLARE_API_TOKEN=your-token
CLOUDFLARE_ACCOUNT_ID=your-account-id
```

| Driver | Best For |
|---|---|
| `cloudflare` | Production — managed headless browser |
| `gotenberg` | Self-hosted headless browser (Docker Compose) |
| `dompdf` | Fallback — pure PHP, no external dependencies |

## Search

Meilisearch is optional. For production or search-heavy local testing:

```ini
SCOUT_DRIVER=meilisearch
MEILISEARCH_HOST=http://127.0.0.1:7700
MEILISEARCH_KEY=your-master-key
```

:::caution
For production, always set a `MEILISEARCH_KEY` and configure the same key as `MEILI_MASTER_KEY` on the Meilisearch container. Without a master key, Meilisearch runs in development mode and exposes all API endpoints without authentication.
:::

## Branding

Runtime branding is loaded from database settings, not compiled into the image. After your first deploy, visit the admin panel to customize:

- Institution name and logo
- Portal colors and themes
- Email templates
- Domain-specific settings

## Health Checks

```bash
# Application-level health endpoint (lightweight, no DB)
curl -s http://localhost:8000/health

# Comprehensive system info
php artisan about

# Route inventory
php artisan route:list --except-vendor
```

For Docker production containers:

```bash
# Container health status (uses built-in HEALTHCHECK)
docker ps --format "table {{.Names}}\t{{.Status}}"

# Detailed health log
docker inspect --format='{{json .State.Health}}' koakademy-app | python3 -m json.tool

# Octane process check
docker exec koakademy-app php artisan octane:status

# Full system overview
docker exec koakademy-app php artisan about
```
