## Docker

### Pulse crashes with `SIGSEGV (core dumped)`

If your logs show `pulse` repeatedly exiting with `SIGSEGV`, the cause is almost always the **PHP 8.5 nightly image** used in the pre-built container. PHP 8.5 is still in development and is prone to segfaults, especially when extensions like `redis`, `opcache`, or `gd` are loaded.

**Quick workaround:**

```ini
# .env
PULSE_ENABLED=false
```

Then restart:

```bash
docker compose down
docker compose up -d
```

**Proper fix:** Build the image with PHP 8.4 (the version `composer.json` requires). Update your `docker/Dockerfile`:

```dockerfile
ARG PHP_VERSION=8.4
```

Then rebuild:

```bash
docker compose build --no-cache app
docker compose up -d
```

### `portal.koakademy.test` returns 502 Bad Gateway

Traefik cannot resolve the upstream `laravel` container. Fix by recreating the services:

```bash
docker compose up -d --force-recreate laravel.test traefik
```

If that doesn't work, restart the full stack:

```bash
docker compose down
docker compose up -d
```

Verify:

```bash
curl -skI https://portal.koakademy.test
```

### Port already in use

If ports `80`, `443`, or `5173` are taken:

```bash
# Check what's using the port
lsof -i :80

# Stop conflicting services or change forwarded ports in .env
FORWARD_DB_PORT=5433
VITE_PORT=5174
```

### Container fails to start

```bash
# Check logs
docker compose logs -f laravel.test

# Rebuild the image
docker compose build --no-cache laravel.test
```

## SSL / HTTPS

### Browser warns about certificate

Regenerate local certificates:

```bash
# macOS / Linux
./scripts/fix-ssl.sh

# Windows
.\scripts\fix-ssl.ps1
```

Then hard-refresh the page (`Ctrl+Shift+R` or `Cmd+Shift+R`).

### `mkcert` not found

```bash
# macOS
brew install mkcert
mkcert -install

# Linux
sudo apt install libnss3-tools
# Download mkcert from https://github.com/FiloSottile/mkcert/releases

# Windows
# Install via Chocolatey: choco install mkcert
```

## Laravel / PHP

### Class not found / Autoload issues

```bash
vendor/bin/sail composer dump-autoload
```

### Key generation

```bash
vendor/bin/sail artisan key:generate
```

### Permission denied on storage

```bash
vendor/bin/sail artisan cache:clear
vendor/bin/sail artisan config:clear
vendor/bin/sail artisan view:clear
```

## Frontend

### Vite manifest error

```bash
vendor/bin/sail npm run build
```

Or start the dev server:

```bash
vendor/bin/sail npm run dev
```

### Node modules issues

```bash
rm -rf node_modules package-lock.json
vendor/bin/sail npm install
```
